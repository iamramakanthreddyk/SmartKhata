## üìù Writing & Updating Documentation

### When to Update Docs

Update immediately when:
- [ ] Architecture changes
- [ ] New feature added
- [ ] API endpoint changed
- [ ] Security requirement added
- [ ] Process or workflow changed

Update monthly:
- [ ] Performance metrics in 11_MAINTENANCE.md
- [ ] Tech debt list in 11_MAINTENANCE.md

Update quarterly:
- [ ] Business metrics in 05_BUSINESS.md
- [ ] Compliance calendar in 10_COMPLIANCE.md

### How to Keep Docs Accurate

**Rule 1**: Single source of truth
- No duplicate sections across documents
- Reference instead of copy

**Rule 2**: Link liberally
- Cross-reference related sections
- Help readers navigate

**Rule 3**: Update in Git
- Every doc change in commit
- PR review for major changes
- Tag for version releases

**Rule 4**: No bloat
- Remove outdated analysis
- Archive old decisions
- Keep to 12 core documents
